# 网页快照分享工具 (Web Snapshot Sharing Tool) - MVP

网页快照分享工具是一个极简的、用于一键保存当前浏览器已渲染网页并生成可分享链接的 MVP 系统。它可以解决“把原本已登录、受限的网页链接发给朋友，而朋友因权限或网络问题无法访问”的痛点。

> [!IMPORTANT]
> **核心安全原则说明**：
> 本工具保存的是用户当前浏览器中**已经显示的网页快照**，不需要知晓用户在原网站的帐号密码。本工具**不会**绕过登录、**不会**破解权限、也**不会**自动在后台抓取用户本身无权访问的内容。

---

## 1. 项目架构与技术栈
项目采用 `pnpm` monorepo 结构，包含三个核心包：
* **`apps/server`**：后端 API，使用 Express + TypeScript + Prisma + SQLite。
* **`apps/web`**：展示端网页，使用 React + Vite。
* **`apps/extension`**：Chrome 浏览器插件，采用 Manifest V3 规范。

主要运行环境：单个 Docker 容器。
* 数据库文件：`/app/data/app.db` (SQLite)
* 文件存储区：`/app/data/storage/` (存放 HTML 文本、纯文本、截图)
* 插件下载区：`/app/apps/server/public/downloads/extension.zip`

---

## 2. 生产环境部署
系统默认生产环境访问地址为：**`http://<your-server-ip>:35128`**。

在具备 Docker 环境的服务器上，可在项目根目录下通过以下命令一键构建并启动容器：

```bash
docker compose up -d --build
```

### 挂载目录说明
容器启动后，会在本地生成 `./data` 目录，并将其挂载至容器的 `/app/data` 目录。
* `./data/app.db` 保存着所有的快照记录元数据。
* `./data/storage/` 保存着快照的 HTML 源码文件、纯文本文件和视口截图。
* 即使容器销毁重建，快照数据依旧持久保存，不受影响。

## 3. Chrome 插件安装步骤
由于本工具处于 MVP 开发者测试阶段，暂未发布至 Chrome 插件商店。请通过以下步骤进行手动加载：

1. **下载插件压缩包**：打开首页 `http://<your-server-ip>:35128`，点击下载 **Chrome 插件 (.zip)**，它会被下载为 `extension.zip`。
2. **解压文件**：在本地将该 zip 压缩包解压到一个专有目录（如 `web-snapshot-extension`）。
3. **打开扩展程序页**：在 Chrome 浏览器地址栏中输入 `chrome://extensions/` 并回车。
4. **开启开发者模式**：勾选页面右上角的 **「开发者模式」** 开关。
5. **加载已解压程序**：点击页面左上角的 **「加载已解压的扩展程序」**，在弹出的文件选择框中选择刚才解压出来的目录。
6. 插件图标显示在浏览器右上角后，即可点击开始使用。

## 4. 本地开发与联调
在本地开发环境下，可按以下步骤启动全栈服务：

### 1) 安装依赖
需要首先安装好 `pnpm` (若未安装，可运行 `npm install -g pnpm`)。然后在项目根目录下执行：
```bash
pnpm install
```

### 2) 启动开发服务
运行以下命令，将同时在后台并行启动前端 Vite 热更新服务 (监听 `http://localhost:5173`) 和后端 Express 服务 (监听 `http://localhost:3000`)：
```bash
pnpm run dev
```

### 3) 插件本地调试
* 打开 `apps/extension` 下的 `manifest.json`，在 Chrome 中加载整个 `apps/extension` 目录。
* 打开插件 Popup 界面，将“后端服务地址”修改为：`http://localhost:3000`。
* 点击保存即可将本地调试快照发送至本地 Express 后端进行测试。

## 5. 使用流程
1. 在浏览器中正常访问任何一个普通网页（例如一篇技术博客、已登录的内部文档页面等）。
2. 确认页面已完全加载，点击浏览器右上角的快照插件按钮。
3. 插件弹窗中会读取当前页面的标题和链接。
4. 点击 **「📸 保存当前网页」** 按钮。
5. 插件会在后台自动完成以下工作：
   * 抓取网页渲染后的 outerHTML 和 InnerText。
   * 捕捉当前视口部分的图像。
   * 将数据安全打包上传至后端，对 HTML 进行安全清洗过滤，并将文件写入持久化磁盘。
6. 上传完成后，插件会展示生成的分享链接，如 `http://<your-server-ip>:35128/s/abc123`。
7. 点击 **「复制分享链接」** 按钮，发送给好友。好友打开链接即可在“阅读内容”、“页面截图”、“纯文本”三个视图间任意切换浏览。

## 6. MVP 限制与明确不做清单
由于本项目已完成重大架构升级，原本 MVP 阶段的诸多限制（如长截图、样式/图片离线化、访问密码等）均已彻底实现。当前版本明确**排除**或暂不考虑的特性如下：
1. **视频与音频物理文件下载保存**（快照仅保存静态展示页面，不转存超大的音视频文件）。
2. **多用户注册与账号隔离体系**（本项目采用轻量级的全局访问密码机制，不适合做面向公众的多用户 SAAS 服务）。
3. **Playwright / Puppeteer 等服务端无头浏览器动态抓取**（快照完全依赖浏览器插件在前端克隆已成功渲染的 DOM，无需在服务端架设高消耗的无头浏览器）。
4. **绕过网站防爬墙与付费阻断**（仅能抓取用户在当前浏览器中已经能合法正常看得到的网页内容）。
5. **Chrome 官方插件商店上架公开发布**。

## 7. 高级功能版本已实现清单
以下高级特性在当前版本中已 **100% 成功开发并经过测试验证**：
* [x] **高还原度 CSSOM 样式序列化提取**：自动将现代网页动态插入内存的 CSS 规则和外链 CSS 样式表下载并序列化，解决排版混乱丢失问题。
* [x] **完整的滚动页面长截图**：插件端集成 `html2canvas` 自动对长网页进行一键无缝长截图生成。
* [x] **快照物理离线化图片采集（前端 Base64 + 后端兜底）**：插件端利用 Background 脚本跨域特权直接在前端将图片转换为 Base64 封包上传，实现 100% 物理离线阅读。
* [x] **访问控制与分享密码拦截**：支持设置全局首页访问密码，以及为单个快照配置独立的查看提取码。
* [x] **物理销毁与定时过期管理**：支持管理员免 deleteKey 物理一键销毁，且系统内置 Scheduler 轮询器定期自动清理并物理磁盘擦除已过期的历史快照。
